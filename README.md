# VISMA

This repository contains data, scripts, and analyses related to the VISMA project, with a specific focus on the characterization of integration sites, nearby mutations, and additional quality-control analyses performed for the point-by-point response.

The repository is organized to keep the input data, core analyses, and additional response-related material clearly separated and reproducible.

------------------------------------------------------------------------

## Repository structure

``` text
VISMA/
├── data/
├── point_by_point/
└── README.md
```

------------------------------------------------------------------------

## `data/`

The `data/` folder contains the main input files used for the analyses.

These files include information about:

- the analyzed samples;
- the integration sites identified by VISMA;
- the genomic annotation of integration sites;
- the mutations detected around integration sites;
- mutation calls obtained from VISMA-based analyses;
- mutation calls obtained from whole-genome sequencing (WGS) analyses.

In particular, this folder collects the core tables needed to link each sample to its corresponding integration sites and mutation profiles. These data are used to compare the mutations detected by VISMA with those identified through WGS, and to evaluate the genomic context of the observed events.

The information stored in this folder represents the starting point for the downstream analyses and figures reported in the project.

------------------------------------------------------------------------

## `point_by_point/`

The `point_by_point/` folder contains additional analyses generated to support the point-by-point response.

The aim of these analyses is to make the production of figures, summary statistics, and quality-control checks transparent and reproducible.

This folder includes:

- the R Markdown report used to generate the additional analyses;
- the input files required by the report;
- the custom plotting theme, if available;
- the output tables generated by the report;
- the output figures generated by the report.

The main report is:

``` text
point_by_point_quality_analyses.Rmd
```

This R Markdown document produces an HTML report describing:

1.  the spatial distribution of mutations relative to the integration site;
2.  the distance between mutations and integration sites;
3.  the comparison of mutation positions across datasets;
4.  the read-end distance distribution of ALT-supporting bases;
5.  the Phred-quality distribution of ALT-supporting bases;
6.  the statistical summaries and tests used to support the interpretation.

The report is intended to document how the plots and statistical results were generated for the point-by-point response.

------------------------------------------------------------------------

## Running the point-by-point analyses

To generate the HTML report, open R or RStudio from inside the `point_by_point/` folder and run:

``` r
rmarkdown::render("point_by_point_quality_analyses.Rmd")
```

Alternatively, from the root of the repository, run:

``` r
rmarkdown::render("point_by_point/point_by_point_quality_analyses.Rmd")
```

The analysis expects the required input files to be available inside the `point_by_point/` folder.

The main required files are:

``` text
mpileup_quality_summary_per_mutation.xlsx
df_phred_wide.xlsx
```

The optional plotting-theme file is:

``` text
ggplot.R
```

If `ggplot.R` is not available, the report uses a default `ggplot2` theme.

------------------------------------------------------------------------

## Output of the point-by-point report

When the R Markdown file is rendered, the report creates a folder named:

``` text
point_by_point_outputs/
```

This folder contains:

``` text
point_by_point_outputs/
├── plots/
└── tables/
```

The `plots/` folder contains the figures generated by the report, including the mutation-distance plots and the sequencing-quality histograms.

The `tables/` folder contains the corresponding summary tables and statistical test results.

------------------------------------------------------------------------

## Reproducibility

The point-by-point analyses are written as an R Markdown report to ensure that the code, figures, statistical tests, and textual explanations are kept together in a single reproducible document.

The final section of the report prints the R session information, including package versions, to facilitate reproducibility across different computational environments.
